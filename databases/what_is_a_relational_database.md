# What is a Relational Database?

A Relational Database is a way of structuring data in rows in columns where there are predefined relationships between fields. 

In this context a **relationship** is a logical connection between **different tables**, and it is based on their interaction. 

Some  examples are:

- `One to Many` → One Case can have Many Users!
- `Many to Many`  → Many mediators can have many cases!
- `One to One` → One User has One Name!
